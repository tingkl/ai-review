# commit-ai-guardian GIF 录制分镜脚本

## 准备工作

### 终端设置
- 字体大小：**16pt+**（方便观众阅读）
- 主题：**深色**（黑色背景，亮色文字）
- 窗口大小：建议 100x30（100列 x 30行）
- 清除所有历史记录，从干净状态开始

### VS Code 设置
- 主题：Dark+
- 字体：推荐 JetBrains Mono / Fira Code
- 字号：14+
- 文件：`example.js`（代码见下方）

### 初始代码（example.js）
```javascript
function getUser(userId) {
  const sql = `SELECT * FROM users WHERE id = ${userId}`;
  return db.query(sql);
}
```

### 修复后代码
```javascript
function getUser(userId) {
  return db.query("SELECT * FROM users WHERE id = ?", [userId]);
}
```

## 分镜脚本（总时长约 30 秒）

### 第 1 镜：安装（0-5 秒）

| 项目 | 内容 |
|------|------|
| **画面** | 终端深色背景 |
| **操作** | 输入安装命令 |
| **文案** | `uv tool install commit-ai-guardian` |
| **动作** | 回车执行，等待安装完成 |
| **提示** | 如果已安装，可以跳过这步，从第 2 镜开始 |

### 第 2 镜：项目初始化（5-9 秒）

| 项目 | 内容 |
|------|------|
| **画面** | 终端 |
| **操作** | 进入项目目录并初始化 |
| **文案** | `cd my-project && cag install` |
| **动作** | 回车，显示 "Git Hook 安装成功" |
| **文案** | `cag configure`（快速展示配置过程） |

### 第 3 镜：展示有问题的代码（9-13 秒）

| 项目 | 内容 |
|------|------|
| **画面** | VS Code 打开 `example.js` |
| **操作** | 鼠标指向 SQL 拼接的那一行 |
| **重点** | 高亮 `${userId}` 这部分 |
| **停留** | 2 秒，让观众看清问题代码 |

### 第 4 镜：Git 提交被拦截（13-20 秒）

| 项目 | 内容 |
|------|------|
| **画面** | 终端，执行 git commit |
| **操作** | `git add example.js && git commit -m "feat: add user query"` |
| **动作** | 回车，等待 AI 审核 |
| **效果** | 屏幕出现审核结果，**红色/黄色**警告 |
| **关键文案** | `⚠️ SQL 拼接存在注入风险` |
| **关键文案** | `💡 使用参数化查询` |
| **关键文案** | `❌ 审核未通过` |
| **停留** | 3 秒，让观众看清审核结果 |

### 第 5 镜：修复代码（20-25 秒）

| 项目 | 内容 |
|------|------|
| **画面** | VS Code，编辑 `example.js` |
| **操作** | 删除有问题的代码，改成参数化查询 |
| **修改前** | `` `SELECT * FROM users WHERE id = ${userId}` `` |
| **修改后** | `"SELECT * FROM users WHERE id = ?", [userId]` |
| **动作** | 保存文件（Ctrl+S） |

### 第 6 镜：重新提交通过（25-30 秒）

| 项目 | 内容 |
|------|------|
| **画面** | 终端 |
| **操作** | `git add example.js && git commit -m "fix: use parameterized query"` |
| **效果** | 绿色 ✅ 通过 |
| **关键文案** | `✅ 审核通过` |
| **关键文案** | `所有文件符合代码质量标准` |
| **停留** | 2 秒，定格在成功画面 |

## 录制技巧

### 鼠标移动
- **慢速移动**，不要一闪而过
- 指向关键代码时**停留 1-2 秒**

### 打字速度
- 命令输入可以**稍快**（观众熟悉 git 命令）
- 但**回车后要等待结果出现**，不要急着下一步

### 关键画面停留时间
| 画面 | 停留时间 |
|------|----------|
| 安装完成 | 1 秒 |
| 问题代码高亮 | 2 秒 |
| AI 审核结果（红色） | 3 秒 |
| 修复过程 | 2 秒 |
| 审核通过（绿色） | 2 秒 |

### 不要录的内容
- 网络等待（太长的 API 调用）
- 密码输入
- 与演示无关的终端输出

## 工具推荐

| 工具 | 平台 | 特点 |
|------|------|------|
| **ScreenToGif** | Windows | 免费，可编辑帧，推荐 |
| **LICEcap** | Win/Mac | 极简，轻量 |
| **Kap** | Mac | 开源，可导出 GIF/MP4 |
| **ShareX** | Win | 功能强大，自带录屏 |

### 导出设置
- 分辨率：**1280x720** 或更高
- 帧率：**10-15 fps**（文件小，够用）
- 文件大小：**控制在 5MB 以内**

```
 5秒     4秒        4秒           7秒         5秒      5秒
```
