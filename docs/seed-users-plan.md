# 种子用户招募方案

## 目标

招募 **10 个团队/个人开发者** 深度试用，收集真实场景反馈，迭代产品。

## 种子用户画像

| 类型 | 特征 | 价值 |
|------|------|------|
| 小团队技术负责人 | 5-15 人，没有专职 CR | 验证"替代人工 CR"的核心价值 |
| 开源项目维护者 | 接受外部 PR，需要质量把关 | 验证自定义案例系统的扩展性 |
| 全栈独立开发者 | 一人多项目，容易遗漏问题 | 验证个人场景的使用体验 |
| 大厂小团队 | 有 CR 但流于形式 | 验证跟现有工作流的整合 |
| 技术博主/KOL | 愿意分享使用体验 | 验证后的口碑传播 |

## 招募渠道

1. **V2EX 帖子评论区** — 直接回复邀请
2. **技术微信群** — 发试用邀请，限 10 人
3. **GitHub Issue** — 开 `good first issue` 标签的试用招募帖
4. **朋友圈/推特** — 个人社交圈扩散
5. **直接私信** — 认识的团队负责人一对一邀请

## 招募文案

```
【AI 代码审查工具内测招募】

找 10 个团队/开发者试用 commit-ai-guardian。

我们能提供：
✅ 免费的 API Key（试用期内，限额 1000 次审核）
✅ 一对一技术支持（微信直接找开发者）
✅ 团队案例定制（帮你把规范写成 AI 案例）

你需要做：
- 安装试用一周
- 每周反馈一次使用感受（好的坏的都要）

感兴趣留言或私信，先到先得。
```

## 反馈收集模板

每周给用户发一个极简问卷：

```
【第 X 周反馈】

1. 这周提交了多少次？AI 拦截了多少次？
   _

2. 拦截的问题中，有多少是真的问题？多少是误报？
   _

3. 最让你惊喜的一个拦截案例是什么？
   _

4. 最让你烦的一个误报案例是什么？
   _

5. 有什么功能你特别想要但还没有的？
   _

6. 1-10 分，整体满意度？
   _
```

## 激励机制

| 阶段 | 激励 |
|------|------|
| 试用满 1 周 | 赠送 5000 次审核额度 |
| 提交有效反馈 | 赠送 10000 次审核额度 |
| 推荐新用户 | 双方各赠送 5000 次 |
| 成为核心用户 | 永久免费 + 名字写入 CONTRIBUTORS |

## 转化目标

- 10 个种子用户中，**7 个以上** 愿意继续使用
- 收集 **30+ 条有效反馈**
- 产出 **5 个真实使用案例**（用于后续推广文章）
- 种子用户自发推荐 **10+ 新用户**
